Overview

This documentation covers the SF Helper and P4V Helper scripts designed to help mods in the content review, uploading, and building process. These tools help avoid mistakes or misclicks showing the most used information and buttons in an overlay for Salesforce, we only use the information showed on screen this doesn't modify anything on SF just make it easy for you to read the information you need in your workflow.

Before Using:

By using these tools you acknowledge the following:

  • You still need to verify that all the steps in your workflow are being executed correctly.
  • These are just tools to save you some time gathering the needed info.
  • You are still responsible for your own work.
  • In case you found a bug or some tool is not working as expected you should stop using the tool and please report the issue to me (Oscar O. in Slack) and I'll try to fix it.

Features
Salesforce Tools

Assign to me

With this button you can easily assign a SF case to you, just click it and while is loading you can click Copy SF case and claim the case also in Slack.

Get Info

Will show you a friendly summary of the app names with YES in the colum of "Is New or Changed" from this window you can easily see the supported versions for each app Name copy the app name (to quickly find it in the Depot) or in case you are working on a new submission or Adding a new version (update ADD case) you can click "P4V data" and then with the other tool "P4Vhelper" you can click "Get data" and from there you can Quickly create a folder in the P4V workspace, this will save you a lot of clicks. There is also a button "Horde" that will quickly create the Job horde for you with just one click.

You can also see in the top of this section the warning color indicator of the seller, Case Type (New submission or Update) and the SF case

Copy SF Case

We always need the SF case number for everything (claiming the case in slack, notes on p4, etc) thats why I've included a dedicated button for this. Just click it and the case number will be copied to your clipboard.

Create TRC

For new submissions review, I've included this button to quickly create a TRC using the Product title and SF Case number, the first time you use any of the templates, the script will ask you to paste the Sheet Template ID.

To get the Template sheet ID, use the following URL as an example: 

https://docs.google.com/spreadsheets/d/1fYkeQQ_bLESF7aw4nswwiw_5ZI7IH6mp8yqIATX39MQ/edit?gid=1812179549#gid=1812179549

The Template sheet ID is the part between `/d/` and `/edit`, which in this case is 
1fYkeQQ_bLESF7aw4nswwiw_5ZI7IH6mp8yqIATX39MQ
This is the ID you need to paste when the script asks for it. After the first time the script will remember the ID and will use it to create the TRCs

The first time the script will also ask you for a Google Drive folder where all these Sheets will be stored, I suggest using your folder from the modsquad ID, this way you can have all the TRCs in one place, you can get your folder ID like this:

Go to the agent folders Drive and enter in the folder with your name, you need to copy the ID of the folder the same way you did with the sheet.

Salesforce is not too friendly when changing status, to avoid sending the wrong status I've included these clean buttons for the most used status, just click the one you need and the status will be changed automatically in just 2 seconds, so make sure you already send the proper email and/or did the full review before clicking these buttons.

Shift Tools

Salesforce is not too friendly when changing status, to avoid sending the wrong status I've included these clean buttons for the most used status, just click the one you need and the status will be changed automatically in just 2 seconds, so make sure you already send the proper email and/or did the full review before clicking these buttons.

Case Log

These buttons will change the status of the case to the selected one, the script will wait 2 seconds before changing the status, this is to avoid sending the wrong status by mistake.

Add shifts / Show Shifts

With this button the script will ask you for your tanda calendar, just go to your Tanda schedule URL (https://my.tanda.co/staff) you must be logged with Okta, click on your name / Attendance points / Schedules / Click on Subscribe iCal / Open that iCal URL, copy all that content, and paste it there.

Now the script will know your shifts and the shift report button will be enabled, if you add more shifts to your schedule then you should "Update shifts" there is a button for that too.

After you saved your calendar you can now enable reminders, so in case you forgot to clock out and you are still in Salesforce working, a small alert will popup with a reminder.

Shift Report

Create automatically a Shift report with the cases registered in Case Log, includes a custom multiplier (in case and update or New submission took a lot of time) also includes a custom event feature so you can substract from the worked time the time you spent in a meeting or any other event like Sketchfab patrol or Horde outages.

Installation

To install the scripts, follow these steps:

  1. Open Tampermonkey in your browser.
  2. Create a new script and paste the content of the desired .js file.
  3. Save the script and make sure Tampermonkey is enabled.

Usage

Ensure that you understand the functionality of each script and that it aligns with the project requirements. Use these scripts responsibly to maintain productivity and efficiency.

SF Helper

Overview

The SF Helper script enhances Salesforce functionality, making it easier to access and visualize the specific information needed for the project.

Features

  • One click Copy SF Case
  • Manage cases with floating buttons
  • Show only the needed information (app names with YES) with "Get info" button
  • Create new TRC with Product title and Case number as a name
  • Use custom templates (sheets ID) to pre-load a TRC for each product type
  • Store and see your workforce shifts
  • Clock-in / out reminders
  • Easily create a shift report with the cases worked

Usage

To use the SF Helper script, follow these steps:

  1. Ensure Tampermonkey is enabled in your browser.
  2. Navigate to Salesforce and the script will automatically enhance the functionality.
  3. Use the floating buttons to manage cases and copy information.

P4V Helper

Overview

The P4V Helper script assists in creating new folders in the depot and automating various tasks related to the content review process.

Features

  • Quickly create new folders in the depot
  • Send needed data to the Horde
  • Clear workspace directories
  • Toggle dark mode

Usage

To use the P4V Helper program, follow these steps:

  1. Download the P4V Helper .exe program.
  2. Run the .exe program on your computer.
  3. Select the same P4V workspace used in the P4V visual client.
  4. Use the GUI to select the workspace, enter the required information, and create folders.
  5. Use the "Get Data" button to load data from the clipboard.
  6. Use the "Clear Workspace" button to clear specific subdirectories.
  7. Toggle dark mode using the button at the bottom.

GSheet Organizer

Overview

The GSheet Organizer script helps organize Google Sheets into specific folders and update sheet titles.

Features

  • Organize Google Sheets into specific folders
  • Update sheet titles

Usage

To use the GSheet Organizer script, follow these steps:

  1. Ensure Tampermonkey is enabled in your browser.
  2. Navigate to Google Sheets and the script will automatically add buttons to organize sheets.
  3. Use the buttons to move sheets to specific folders and update titles.

Troubleshooting

If you encounter any issues while using the scripts, try the following steps:

  • Ensure Tampermonkey is enabled in your browser.
  • Check for any errors in the browser console (press F12 to open the console).
  • Ensure you have the latest version of the scripts.
  • Contact the author for further assistance.
